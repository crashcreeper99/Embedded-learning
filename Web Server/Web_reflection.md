# Project Reflection – ESP32 Web Server

Completing the ESP32 Web Server project helped me understand how embedded systems can communicate over a network. Before this project, I viewed microcontrollers mainly as devices that controlled hardware directly. Building a web server showed me that an ESP32 can also act as a networked device capable of serving webpages and responding to requests from other computers and smartphones.

One of the most valuable lessons was learning how the ESP32 connects to a Wi-Fi network and receives an IP address that allows other devices to communicate with it. I also gained a better understanding of HTTP requests, client-server communication, and how web technologies can be integrated with embedded programming.

I encountered a few challenges while testing the web server, including verifying the network connection and accessing the correct IP address. Troubleshooting these issues strengthened my debugging skills and increased my confidence in solving networking problems.

This project is an important milestone because it introduces concepts used in Internet of Things (IoT) systems. In future projects, I plan to expand this web server by adding remote LED control, sensor monitoring, and real-time data visualization. These skills will help prepare me for internships and careers involving embedded systems, IoT, robotics, and mechatronics.

Overall, this project improved both my programming and networking knowledge while reinforcing the importance of combining hardware and software to build connected systems.
